To solve the bottleneck in your file processing script, I recommend switching from a sequential approach to a thread-based or process-based concurrent model. Since file I/O can be blocking, concurrent.futures is your best friend here.

Here is a refined implementation using ThreadPoolExecutor. This allows multiple file transfers to happen in parallel without waiting for the previous one to complete.

import shutil
from concurrent.futures import ThreadPoolExecutor
import os

def move_file(file_path, destination):
    try:
        shutil.copy2(file_path, destination)
        # Logging success for technical audit
        return f"Success: {os.path.basename(file_path)}"
    except Exception as e:
        return f"Error: {e}"

def bulk_process(source_list, dest_dir):
    with ThreadPoolExecutor(max_workers=8) as executor:
        results = executor.map(lambda f: move_file(f, dest_dir), source_list)
    return list(results)
